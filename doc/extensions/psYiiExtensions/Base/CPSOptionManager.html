Class CPSOptionManager

Class Overview
Class Overview | Property Summary | Properties Detail | Method Summary | Methods Detail | Constants Summary | Constants Detail

CPSOptionManager provides the base class for generic options settings for use with any class.

Avoids the need for declaring member variables and provides convenience magic functions to search the options.

Here is an example option specification and pattern.

  1.  array(
  2.         //    optionName is the name of your option key, optionally private (by appending an
  3.          //    underscore ('_') to the option name. Private options are never returned from
  4.          //    (@link makeOptions())
  5.          'optionName{_}' => array(
  6.              //    Option name for a javascript widget. Defaults to '<b>optionName</b>'.
  7.             //    Defaults to null,
  8.              CPSOptionManager::META_DEFAULTVALUE => default value,
  9.              //    Lay out the validation parameters
  10.              CPSOptionManager::META_RULES => array(
  11.                  //    Any valid PHP type (i.e. string, array, integer, etc.),
  12.                  'type' => 'typename',
  13.                  '_extName' => external-facing name | empty
  14.                  // An array of valid values for the option
  15.                  'valid' => array( 'v1', 'v2', 'v3', etc.)
  16.          )
  17.  )

_extName tells the option manager to use this when formatting an options array with (@link CPSOptionManager::makeOptions()).

_value tells the option manager what to set the value of the option to upon creation. New values will override this default value.

When adding options to the options manager, you must specify the pattern for the option's value. The pattern must be placed in the array element with a key of '_validPattern' to be recognized by the option manager.

_validPattern tells the option manager what type of variable is allowed to be assigned to the option. Any legal PHP type is allowed. If more than one type is allowed, you may send in an array as the value of 'type'.

This pattern can include none, one, or more pattern sub-types.

These are:

    2. valid 3. required

    type tells the option manager what type of variable is allowed to be assigned to the option. Any legal PHP type is allowed. If more than one type is allowed, you may send in an array as the value of 'type'.

    valid tells the option manager the valid values of the option that is being set. This must be specified as an array. For instance, if an option can only have three possible values: 'public', 'protected', or 'private', the array specified for the value of 'valid' would be

    required tells the option manager that the option is required and as such, must have a non-null _value.

    1.  array( 'public', 'protected', 'private' )

    This next snippet defines an option named 'hamburgerCount'. It is private because we've appended an underscore to the name. It's default value is 6, it can be only of type 'int', and has no required values.

    1.  'hamburgerCount_' = array(
    2.      '_value' => 6,
    3.      '_validPattern' =>
    4.          array(
    5.              'type' => 'int',
    6.              'valid' => null,
    7.          ),
    8.      );

    Using this option from a Pogostick component or widget is as simple as this:

    1.  $this->hamburgerCount_ = 3;
    2.  echo $this->hamburgerCount_;
