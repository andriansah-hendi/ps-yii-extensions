Class CPSOptionManager

Description
Description | Vars (details) | Methods (details) Constants (details)

CPSOptionManager provides the base class for generic options settings for use with any class.

Avoids the need for declaring member variables and provides convenience magic functions to search the options.

Here is an example option specification and pattern.

  1.  array(
  2.         //    optionName is the name of your option key, optionally private (by appending an
  3.          //    underscore ('_') to the option name. Private options are never returned from
  4.          //    (@link makeOptions())
  5.          'optionName{_}' => array(
  6.              //    Option name for a javascript widget. Defaults to '<b>optionName</b>'.
  7.             //    Defaults to null,
  8.              CPSOptionManager::META_DEFAULTVALUE => default value,
  9.              //    Lay out the validation parameters
  10.              CPSOptionManager::META_RULES => array(
  11.                  //    Any valid PHP type (i.e. string, array, integer, etc.),
  12.                  CPSOptionManager::META_TYPE => 'typename',
  13.                  //    Is option required?
  14.                  CPSOptionManager::META_REQUIRED => true|false,
  15.                  //    The external name of the option (i.e. what to send to component)
  16.                  CPSOptionManager::META_EXTERNALNAME => external-facing name | empty
  17.                  // An array of valid values for the option
  18.                  CPSOptionManager::META_ALLOWED => array( 'v1', 'v2', 'v3', etc.)
  19.          )
  20.  )

CPSOptionManager::META_EXTERNALNAME tells the option manager to use this when formatting an options array with (@link CPSOptionManager::makeOptions()).

CPSOptionManager::META_DEFAULTVALUE tells the option manager what to set the value of the option to upon creation. New values will override this default value.

When adding options to the options manager, you must specify the pattern for the option's value. The pattern must be placed in the array element with a key of 'CPSOptionManager::META_RULES' to be recognized by the option manager.

This pattern can include none, one, or more pattern sub-types.

These are:

    2. CPSOptionManager::META_ALLOWED 3. CPSOptionManager::META_REQUIRED

    CPSOptionManager::META_TYPE tells the option manager what type of variable is allowed to be assigned to the option. Any legal PHP type is allowed. If more than one type is allowed, you may send in an array as the value of 'CPSOptionManager::META_EXTERNALNAME'.

    CPSOptionManager::META_ALLOWED tells the option manager the valid values of the option that is being set. This must be specified as an array. For instance, if an option can only have three possible values: 'public', 'protected', or 'private', the array specified for the value of 'CPSOptionManager::META_TYPE' would be:

    1.  array( 'public', 'protected', 'private' )

    CPSOptionManager::META_REQUIRED tells the option manager that the option is required and as such, must have a non-null value.

    This next snippet defines an option named 'hamburgerCount'. It is private because we've appended an underscore to the name. It's default value is 6, it can be only of type 'integer', and has no required values.

    1.  'hamburgerCount_' = array(
    2.      CPSOptionManager::META_DEFAULTVALUE => 6,
    3.      CPSOptionManager::META_RULES =>
    4.          array(
    5.              CPSOptionManager::META_TYPE => 'integer',
    6.              CPSOptionManager::META_ALLOWED => null,
    7.          ),
    8.      );

    Using this option from a Pogostick component or widget is as simple as this:

    1.  $this->hamburgerCount = 3;
    2.  echo $this->hamburgerCount;

    Once you declare an option private (suffixing with the underscore as above, you no longer need to provide the underscore when accessing the option. The underscore is used ONLY when adding new options and is dropped once added.

			$m_sPrefixDelimiter
			 =  '::'			(line 207)
		
	
    

	

    The delimiter to use for prefixes. This must contain only characters that are not allowed  in variable names (i.e. '::', '||', '.', etc.). Defaults to '::'. There is no length limit,  but 2 works out. There is really no need to ever change this unless you have a strong dislike  of the '::' characters.
