string $m_sInternalName (line 49)

The internal name of the component. Used as the name of the behavior when attaching.

In order to facilitate option separation, this value is used along with the prefix delimiter by the internal option manager to distinguish between owners. During construction, it is set to the name of the class, and includes special behavior for Pogostick classes. Example: CPSComponent becomes psComponent. Use or override (@link setInternalName) to change the name at runtime.

  • see: CPSWidget::setInternalName()

The delimiter to use for prefixes. This must contain only characters that are not allowed in variable names (i.e. '::', '||', '.', etc.). Defaults to '::'. There is no length limit, but 2 works out. __get (line 286)

Returns a property value, an event handler list or a behavior based on its name.

Do not call this method. This is a PHP magic method that we override to allow using the following syntax to read a property or obtain event handlers:

  1.  $value=$component->propertyName;
  2.  $handlers=$component->eventName;

Will also return a property from an attached behavior directly without the need for using the behavior name

  1.  $value = $component->behaviorPropertyName;
instead of
  1.  $value = $component->behaviorName->propertyName

  • return: the property value, event handlers attached to the event, or the named behavior
  • see: CPSWidget::__set()
  • throws: CException if the property or event is not defined
  • access: public
mixed __get (string $sName)
  • string $sName: the property name or event name
